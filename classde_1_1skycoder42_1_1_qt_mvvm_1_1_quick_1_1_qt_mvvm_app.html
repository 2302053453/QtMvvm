Detailed Description

An application root window that is a full fledged QML presenter.

Since
1.0

This is the standard presenter you should use in your mvvm application as the root QML component of the primary qml file loaded by the engine. It will automatically register as the main QML presenter and can handle all the standard view types.

See also
PresenterProgress, PresentingStackView, PresentingDrawer, PopupPresenter, DialogPresenter, QuickPresenter

◆ closeAction()

bool de::skycoder42::QtMvvm::Quick::QtMvvmApp::closeAction ( )

Can be called to perform a close operation.

Returns
true if anything was closed, false if not

Use this method to perform a close operation. The app will automatically check all internally used presenter in a logical order and call this method on all of them until one returns with a positive result. This way you have a global method to close any kind of active view. This is also used to handle window close events and the mobile back button.

◆ rootOnlyDrawer

bool de::skycoder42::QtMvvm::Quick::QtMvvmApp::rootOnlyDrawer

Specifies if the drawer can be accessed from the root element only.

Default: true

If set to true, the presenter can only be accessed by the user if only one view is currently presented on the internal PresentingStackView. As soon as second view gets pushed, the user cannot open the drawer anymore. By settings it to false, the drawer will always be accessible on any view on the stack.
